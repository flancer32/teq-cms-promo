ADSM: managing a site with an agent

What and Why

ADSM (Agent Driven Site Management) lets you maintain a site through plain-text instructions. You build the product while the agent keeps the HTML in sync.

Advantages

  • Updates without code or admin panels
  • The site always matches the product description
  • Edit history in git
  • LLM keeps translations in sync

How It Works

  1. You write context and tasks
  2. The agent creates or updates pages
  3. Git stores the changes
  4. The site publishes the latest version

What's Included

  • The ADSM methodology
  • Context and instruction templates
  • Example HTML pages
  • A guide to using the agent

Who It's For

  • Anyone tired of waiting for a developer's help
  • Teams with rapidly changing products
  • Developers working with git and LLM
